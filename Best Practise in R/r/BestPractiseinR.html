Best Practise in R

R is an open source ecosystem with vast array of different packages from many different authors, evolving of over many years and across multiple different disciples. There are no agreed conventions, even within base R.

GABurns

This makes it difficult for a R user to know what convention to adopt.

Aim: To stimulate discussion on best practise, signpost to resources & suggest some approaches to take.

Key messages

The key take home messages from the Jumping Rivers course were:

  • Write code for humans to understand, not computers
  • Best Practise is subjective and specific to individual use-cases
  • Adopt conventions, document them and be consistent within a Team
  • Adopting best practice may not save you time short term but will save time in long term

Naming Conventions

Code will run without a standard naming convention but it will make it difficult for others to read and interpret the code. It can lead to bugs, difficulty to communicate what the code does and inefficiencies.
Some things we name in R: - Files - Functions - Arguments - Variables - Classes

Style Guidelines

  • Put space after a comma and not before
  • Put space before and after an equals sign
  • Use assignment operator (<-) for created functions and variables
  • Maximum 80 characters per line
  • Indent curly brackets (RStudio: Crtl + Shift + A)

Lessons Learnt 3

Create a script to carry out as much tasks as possible or document where externalities occur. Ideally if you’re obtaining external data, write a script for sourcing the data and save it in the same project file structure. If the data was obtained via USB, e-mail or one off transfer - document this. As the next person coming along will have no way of knowing where these data were obtained from!

Lessons Learnt 4

When creating an index for referencing (such as a loop or lapply), use seq_len. Other sequence index creating methods can create a sequence backwards and make it difficult to debug loops.

negativeNumber <- -4
zero <- 0

1:negativeNumber
## [1]  1  0 -1 -2 -3 -4
1:zero
## [1] 1 0
seq(negativeNumber)
## [1]  1  0 -1 -2 -3 -4
seq(zero)
## [1] 1 0
# These will return an error before your loop is run
seq_len(negativeNumber)
seq_len(zero)

Lessons Learnt 5

Use TRUE & FALSE not T & F. TRUE & FALSE are protected terms in R. T & F are not.

T <- FALSE

# NA's propagate
mean(penguins$bill_length_mm)
## [1] NA
# Change the na.rm argument to TRUE by using shorthand T
mean(penguins$bill_length_mm, na.rm = T)
## [1] NA
isFALSE(T)
## [1] TRUE
mean(penguins$bill_length_mm, na.rm = TRUE)
## [1] 43.92193
# TRUE is a protected term in R
TRUE <- FALSE

Lessons Learnt 6

Running a script will automatically overwrite your output files! If you copy and paste a script and start to modify it this can lead to unintended consequences! A simple wrapper function can check if a file exists and if so append a date stamp suffix to prevent overwriting.
